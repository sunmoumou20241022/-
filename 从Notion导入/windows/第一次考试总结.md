## 📚 DNS服务配置与批量区域管理

### 🔧 正反向解析配置与未知域名映射

### 💻 命令行方法 (CLI Method)

在 **dc1** 的 PowerShell 中运行以下配置命令，快速拉起反向查找区域并通过建立根 `.` 区域及星号 `*` 泛解析实现未知域名劫持：

```powershell
Add-DnsServerPrimaryZone -Name "10.168.192.in-addr.arpa" -ZoneFile "10.168.192.in-addr.arpa.dns"
Add-DnsServerResourceRecordA -ZoneName "skills2026.com" -Name "dc1" -IPv4Address "192.168.10.30"
Add-DnsServerResourceRecordPTR -ZoneName "10.168.192.in-addr.arpa" -Name "dc1.skills2026.com" -NetworkAddress "192.168.10.30"

# ⚠️ 核心：拦截任何未知域名的未知记录解析为本机 IP
Add-DnsServerPrimaryZone -Name "." -ZoneFile "root.dns"
Add-DnsServerResourceRecordA -ZoneName "." -Name "*" -IPv4Address "192.168.10.30"
```

### 🖱️ GUI 模式方法 (GUI Method)

1. 打开 **DNS 管理器** (`dnsmgmt.msc`)
2. 右键“反向查找区域” → **新建区域** → 依向导填入网络 ID `192.168.10`
3. 在正向区域 `skills2026.com` 里右键新建主机（A 记录），名称写 `dc1`，IP 填 `192.168.10.30` 并勾选“创建相关的指针(PTR)记录”
4. 右键“正向查找区域” → **新建区域** → 区域名称直接填一个小数点 `.`
5. 建完后右键点击这个 `.` 区域 → **新建主机** → 名称填星号 ，IP 地址填 `192.168.10.30`
6. 在.区域里加入主机*
7. 需要允许区域传送

### 📦 批量创建 [skills1.cn](http://skills1.cn/) 至 [skills100.cn](http://skills100.cn/)

### 💻 命令行方法 (CLI Method)

在 PowerShell 中利用循环管道线，一秒钟批量灌入 100 个权威区域和对应的全解析 A 记录：

```powershell
1..100 | ForEach-Object {
    $zone = "skills$_.cn"
    Add-DnsServerPrimaryZone -Name $zone -ZoneFile "$zone.dns"
    Add-DnsServerResourceRecordA -ZoneName $zone -Name "@" -IPv4Address "192.168.10.30"
}
```

### CMD

```bash
for /l %a in (1,1,100) do dnscmd /zoneadd skills%a.cn /primary
```

**各部分含义：**

- **`for /l`**：$\color{blue}{\textbf{数值范围循环}}$参数，表示按指定步长遍历数值区间
- **`%a`**：循环变量（批处理脚本中需写为 `%%a`，命令行直接执行用 `%a`）
- **`(1,1,100)`**：范围参数 `(起始, 步长, 结束)` → 从 1 开始，每次 +1，直到 100
- **`do`**：对每个值执行后续命令
- **`dnscmd`**：$\color{blue}{\textbf{Windows DNS 服务器管理命令行工具}}$（需安装 DNS 服务器角色）
- **`/zoneadd`**：dnscmd 的子命令，用于添加新 DNS 区域
- **`skills%a.cn`**：区域名称，`%a` 会被替换为当前数字（如 `skills1.cn`、`skills2.cn`...`skills100.cn`）
- **`/primary`**：$\color{green}{\textbf{指定创建主区域}}$（primary zone）

### 🔍 批量添加 A 记录命令

```bash
for /l %a in (1,1,100) do dnscmd /recordadd skills%a.cn @ A 192.168.44.132
```

**各部分含义：**

- **`/recordadd`**：dnscmd 的子命令，用于添加 DNS 记录
- **`skills%a.cn`**：目标区域名称（同上，循环替换）
- **`@`**：$\color{blue}{\textbf{表示区域根域名}}$（即 `skills%a.cn` 本身）
- **`A`**：记录类型为 A 记录（IPv4 地址映射）
- **`192.168.44.132`**：$\color{green}{\textbf{要绑定的 IP 地址}}$

### ⚠️ 重要注意事项

- $\color{red}{\textbf{执行前需确保已安装 DNS 服务器角色}}$，否则 `dnscmd` 命令不可用
- $\color{red}{\textbf{批量操作会创建 100 个独立区域}}$，请确认服务器资源是否充足
- 若在批处理脚本（`.bat`/`.cmd`）中使用，$\color{red}{\textbf{变量需写为 %%a}}$（双百分号）

- 保持路径默认直至配置完成

### 💻 命令行方法 (CLI Method)

在 `dc2` 的 CMD 中执行以下命令，快速构建好完全符合 Windows 活动目录用户隔离标准的树状物理架构并生成答题卡指定的测试文件：

```
md C:\\ftp\\skills2026\\IT1
echo IT1.txt > C:\\ftp\\skills2026\\IT1\\IT1.txt
md C:\\ftp\\skills2026\\IT2
echo IT2.txt > C:\\ftp\\skills2026\\IT2\\IT2.txt
md C:\\ftp\\LocalUser\\Public
```

### 🖱️ GUI 模式方法 (GUI Method)

- **创建站点**：打开 IIS 管理器，右键“网站” -> 选择“添加 FTP 站点”。输入名称 `skills_ftp`，物理路径指向顶层根目录 `C:\\ftp`。
- **绑定与授权**：绑定 IP 设为本地 `192.168.10.40`，SSL 选“允许”。身份验证同时勾选 “匿名” 和 “基本”。授权部分：允许访问下拉选“所有用户”，权限同时勾选 “读取” 和 “写入”。
- **配置隔离与横幅**：左侧树状图点击刚才建立的 FTP 站点图标：
    1. 双击进入 “FTP 用户隔离” -> 选中 “用户名目录 (禁用全局虚拟目录)” -> 点击右上角应用。
    2. 双击进入 “FTP 消息” -> 在横幅 (Banner) 框中输入 `欢迎访问 FTP服务器` -> 点击右上角应用。

### ✅ 验证方法 (Verification)

按照答题卡的标准，依次在对应机器上执行以下三项终极检验：

1. 在 `dc2` 中运行：执行 `tree C:\\ftp /f`，观察并截图输出的树状图。应该能清晰看到 `C:\\ftp\\skills2026\\IT1\\IT1.txt` 等层级关系。
2. 在 `dc2` 中运行：执行 `ftp 192.168.10.40`，在提示输入用户名前，屏幕上应当成功弹出一行 Banner 消息：`220-欢迎访问 FTP服务器`。
3. 在 `client` 客户机上运行：打开 CMD 执行 `ftp 192.168.10.40`，当提示输入用户名时，输入带域名后缀的完整账号：`skills2026\\IT2`，密码输入 `Skills39!`。成功登录后键入 `dir` 命令，回显中只能看到 `IT2.txt` 这一个文件，$\color{red}{\textbf{无法向上越级或横向跨越到 IT1 的专属空间}}$，证明 $\color{blue}{\textbf{FTP 活动目录隔离完美生效}}$！
