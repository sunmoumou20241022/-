# windows Server 2019 创建域

---

### 命令与配置 (Commands / Config - GUI 步骤)

**第一阶段：基础环境准备（必须先做）**

1. 打开 **服务器管理器 (Server Manager)**，点击左侧的 **本地服务器 (Local Server)**。
2. 点击当前随机生成的**计算机名**，在弹出的系统属性中点击“更改”，将计算机名修改为 `sun`。**点击确定，但先选择“稍后重启”**。
3. 点击以太网旁边的 **IPv4 地址**（或者网络连接图标），右键网卡选择**属性** -> 双击 **Internet 协议版本 4 (TCP/IPv4)**。
4. 选择“使用下面的 IP 地址”，填入你的静态 IP（例如：`192.168.10.10`）和子网掩码。
5. **极其关键的一步**：在“首选 DNS 服务器”中，填入 `127.0.0.1`。
6. 手动重启虚拟机，让计算机名 `sun` 生效。

**第二阶段：安装 AD DS 角色**

1. 重启后再次打开 **服务器管理器**。
2. 点击右上角的 **管理 (Manage)** -> **添加角色和功能 (Add Roles and Features)**。
3. 连续点击 **下一步**，直到到达 **服务器角色 (Server Roles)** 界面。
4. 勾选 **Active Directory 域服务 (Active Directory Domain Services)**。此时会弹出一个窗口提示添加所需的功能，点击 **添加功能 (Add Features)**。
5. 连续点击 **下一步**，最后点击 **安装 (Install)**。等待进度条走完（无需关闭窗口）。

**第三阶段：提升为域控制器**

1. 安装完成后，在刚才界面的进度条下方，或者服务器管理器右上角的**黄色警告小旗帜**处，点击蓝色的字：**“将此服务器提升为域控制器 (Promote this server to a domain controller)”**。
2. **部署配置**：选择第三个选项 **“添加新林 (Add a new forest)”**。在**根域名**框中输入 `sun1.local`，点击下一步。
3. **域控制器选项**：林功能级别保持默认（Windows Server 2016）。在下方输入两次 **DSRM（目录服务还原模式）密码**（例如设置一个强密码：`AdminPass123!`，请务必记住）。点击下一步。
4. **DNS 选项**：如果出现“无法创建 DNS 委派”的黄色警告，**直接忽略**，点击下一步。（因为我们是第一台机器，还没有父域）。
5. **其他选项**：向导会自动生成 NetBIOS 域名（应该会自动显示为 `SUN1`）。确认无误后点击下一步。
6. **路径**：保持数据库、日志文件和 SYSVOL 的默认路径（位于 `C:\Windows` 下），点击下一步。
7. **查看选项与先决条件检查**：确认所有配置后，系统会进行先决条件检查。只要界面上方出现**绿色的勾**（提示所有先决条件检查通过），点击 **安装 (Install)**。
8. 系统会自动开始配置，完成后**会自动强制重启**。

### 5. 验证方法 (Verification)

系统重启后，登录界面会发生变化，你应该看到以 `SUN1\Administrator` 身份登录的提示。登录后，按 `Win + R` 打开运行窗口，输入 `cmd`，执行以下验证：

**验证测试：域控身份状态**

- **输入命令**：**`*netdom query fsmo*`**
- **预期成功结果**：终端会输出 5 行信息，每一行都指向 `sun.sun1.local`。这证明你的机器已经成功接管了该域的所有五大核心角色。
- **可能失败特征**：提示 `命令无法执行` 或找不到域控制器。这通常是因为之前网卡的 DNS 没有指向 `127.0.0.1`，导致 DNS 注册失败。

**验证测试：系统全名**

- **输入命令**：`*hostname*`
- **预期成功结果**：输出 `sun`。
- **输入命令**：`*ipconfig /all*`
- **预期成功结果**：在输出信息的最上方，“主 DNS 后缀”会显示为 `sun1.local`。

### 6. 注意事项 (Notes)

- **假设声明 (ASSUMPTION)**：我假设你使用的是全新的、纯净的 Windows Server 2019 虚拟机镜像。如果在执行步骤前，这台机器已经做过其他的 IIS 或网络实验，可能会导致端口冲突或先决条件检查失败。
- GUI 向导在“先决条件检查”阶段可能会报出几条黄色的警告（例如涉及 NT 4.0 兼容性等），这是正常的工程现象，只要最终显示的是“通过了所有检查”，就可以放心点击安装。只有红色的错误才会阻断部署。

**应急突发预案：三步排错法**
如果以后在加域时，再次看到 `DNS 名称不存在 / 0x0000232B` 的报错弹窗，不需要重装系统，直接执行以下**故障域隔离管道** ：  **步骤执行位置核心操作 (GUI 或 命令行)为什么这样做？ (工程逻辑)1. 强刷源头域控制器 (DC)**打开 CMD 运行：
`ipconfig /registerdns`
`net stop netlogon && net start netlogon`强迫域控重新把自己的 LDAP/Kerberos 服务凭证敲黑板写入 DNS 数据库，补齐缺失的 SRV 记录。**2. 清理管道待加域客户端**打开 CMD 运行：
`ipconfig /flushdns`擦除客户端本地因为刚才加域失败而卡死的“错误解析缓存”，防止它继续执迷不悟。**3. 精准验证待加域客户端**打开 CMD 运行：
`nslookup -type=srv _ldap._tcp.dc._msdcs.你的域名`在点图形界面之前进行最终验证，只要能吐出域控的主机名，加域 100% 成功 。
