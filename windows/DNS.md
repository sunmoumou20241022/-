# DNS

在整个复杂的网络实验中，**DNS（域名系统）是连接客户端与各大业务服务器的“引路人”**。如果 DNS 这一步配置错误，客户端连服务器的 IP 都找不到，后续的 HTTPS 重定向和 FTP 用户隔离就根本无从谈起。

以下是针对我们本次实验中 DNS 部分的全面总结，包含完整的操作复盘和高频踩坑点。

## 一、 DNS 的核心配置步骤

在活动目录（AD）网络环境中，所有的域名解析记录都集中托管在域控制器（DC）上。

### 1. 服务器端操作（在域控 `sun` 上）

- **打开工具**：在服务器管理器中点击右上角 **“工具” ➔ “DNS”**（或者按 `Win + R` 输入 `dnsmgmt.msc`）。
- **定位区域**：在左侧展开 DNS 服务器 ➔ 展开 **“正向查找区域” (Forward Lookup Zones)** ➔ 点击进入我们的域名 **`skills.com`**。
- **新建 A 记录（建立指针）**：
    1. 在右侧空白处点击**右键**，选择 **“新建主机 (A 或 AAAA)...”**。
    2. **名称 (N)**：输入子域名 **`ftp`**（此时下方会自动组合出全称 `ftp.skills.com`）。
    3. **IP 地址 (I)**：精准填写承载 FTP 服务的第二台服务器 IP：**`192.168.100.2`**。
    4. 点击 **“添加主机” (Add Host)**，随后点击确定完成。
    *(同理，Web 服务器的 `web` 记录也是这样建立并指向对应 IP 的)*

### 2. 客户端操作与验证（在 Win10 客户端上）

- **对齐网卡参数**：进入网卡属性（TCP/IPv4），将 **“首选 DNS 服务器”** 强行修改为域控的 IP：**`192.168.100.1`**。
- **测试解析（验收）**：打开 CMD 窗口，运行：DOS
    
    ```
    ping ftp.skills.com
    ```
    
    **成功标志**：命令行不能提示“找不到主机”，必须清晰返回：`正在 Ping ftp.skills.com [192.168.100.2]`。只要中括号里的 IP 对了，就说明 DNS 完美过关。
    

## 二、 关键注意事项（防翻车指南）

在实际工程和网络竞赛中，DNS 是最容易让人抓狂的地方，请务必牢记以下核心要点：

### 1. 客户端的 DNS 指向必须“绝对纯净”

- **痛点**：很多同学在做实验时，客户端既想上外网，又想做内网实验，于是在 DNS 里填写了 `192.168.100.1`（域控），备用 DNS 填了 `8.8.8.8`（公网）。
- **后果**：Windows 的解析机制带有随机性。一旦流量跑去了 `8.8.8.8`，公网根本不可能知道你自定义的 `ftp.skills.com` 是什么，就会秒回“找不到主机”。
- **铁律**：**内网实验环境下，客户端的 DNS 只能且必须唯一指向域控 IP！**

### 2. 区分“找不到主机”与“请求超时”

在排错时，通过 Ping 的返回信息可以瞬间定位故障域：

- ❌ **找不到主机 (Host Not Found)**：**100% 是 DNS 的锅**。说明 DNS 服务器里根本没有这条 A 记录，或者客户端找错了 DNS 服务器。此时**不需要**去检查 FTP 服务或防火墙，盯着 DNS 改就行。
- ⚠️ **请求超时 (Request Timed Out)**：**DNS 已经成功了**（因为已经解析出了中括号里的 IP），但流量在路上断了。此时应该去检查**目标服务器的防火墙、网卡连接状态（是否都在 VMnet8）或者服务有没有开启**。

### 3. 主机名填写的“留白”艺术

- 在 DNS 管理器里新建主机（A记录）时，在“名称”一栏**只需要写前缀（如 `ftp` 或 `web`）**。
- 千万不要手抖在里面写成了全称 `ftp.skills.com`，否则系统在合并后缀后，你的域名就会变成极其奇葩的 `ftp.skills.com.skills.com`，导致解析彻底失效。

### 4. 遭遇修改不生效？善用“冲刷缓存”

有时候你在服务器上修改了 DNS 记录，但在 Win10 客户端 Ping 出来的依然是旧 IP，这是因为操作系统本地有 DNS 缓存。

- **绝杀命令**：在 Win10 客户端的 CMD 中运行：DOS
    
    ```
    ipconfig /flushdns
    ```
    
    这行命令会瞬间排空本地所有的域名老缓存，强迫网卡重新去域控拉取最新的 A 记录。
    

**总结成一句话**：做域网络实验，**先对齐 Client 的 DNS 填域控，再去域控的 DNS 里建 A 记录指向业务机**。只要遵循这个自底向上的逻辑，网络解析这一层你将永远立于不败之地！
