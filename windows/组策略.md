# 组策略

[组策略.docx](%E7%BB%84%E7%AD%96%E7%95%A5/%E7%BB%84%E7%AD%96%E7%95%A5.docx)

这是一份为您量身定制的 **Windows Server 2019（主/副域控）** 与 **Windows 10 x64（客户端）** 真实虚拟机实验环境的 **全套 9 个步骤一站式通关总结大礼包**。

每个步骤均包含 **图形界面（GUI）** 与 **命令行（CLI）** 两种方法，明确指出**最简方法**，并附带最直观的 **📸 验证与高分截图指南**，方便您直接截取放进实验报告或作业中。

### 步骤 1：取消所有计算机的交互式登录按 Ctrl+Alt+Del 的功能

- **图形界面法（GUI）**：在主域控打开组策略管理（`gpmc.msc`），右键编辑 `Default Domain Policy`。导航到：`计算机配置` → `策略` → `Windows 设置` → `安全设置` → `本地策略` → `安全选项`。双击 `交互式登录: 无须按 CTRL+ALT+DEL`，设置为 **“已启用”**。
- **命令行法（CLI）**：在主域控的 PowerShell 中运行：PowerShell
    
    ```
    Set-GPRegistryValue -Name "Default Domain Policy" -Key "HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\System" -ValueName "DisableCAD" -Type DWord -Value 1
    ```
    
- **💡 最简选择**：**GUI 法最简单**，安全选项路径固定，不易输错注册表。
- **📸 验证与截图**：在 Windows 10 客户端运行 `gpupdate /force`，按 `Win + L` 锁屏。再次唤醒屏幕时，**界面直接显示密码输入框，没有提示“请按 Ctrl+Alt+Del 开始”**。截取此登录画面。

### 步骤 2：实现所有域用户登录服务器时不自动显示服务器管理器

- **图形界面法（GUI）**：在上述组策略编辑器中导航到：`计算机配置` → `策略` → `管理模板` → `系统` → `服务器管理器`。双击 `在登录时不自动显示服务器管理器`，设置为 **“已启用”**。
- **命令行法（CLI）**：在主域控的 PowerShell 中运行：PowerShell
    
    ```
    Set-GPRegistryValue -Name "Default Domain Policy" -Key "HKLM\Software\Policies\Microsoft\Windows\ServerManager" -ValueName "DoNotOpenServerManagerAtLogon" -Type DWord -Value 1
    ```
    
- **💡 最简选择**：**GUI 法最简单**，Server 2019 管理模板汉化完整，极易勾选。
- **📸 验证与截图**：使用任意域用户登录副域控（Server 2019），进入桌面后 **服务器管理器保持静默、没有自动弹窗**。截取干净的服务器桌面。

### 步骤 3：域用户登录云主机（Windows 10）时显示上一次尝试登录的交互信息

- **图形界面法（GUI）**：在组策略编辑器中导航到：`计算机配置` → `策略` → `管理模板` → `Windows 组件` → `Windows 登录选项`。双击 `在用户登录期间显示有关以前登录的信息`，设置为 **“已启用”**。
- **命令行法（CLI）**：在主域控的 PowerShell 中运行：PowerShell
    
    ```
    Set-GPRegistryValue -Name "Default Domain Policy" -Key "HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\System" -ValueName "DisplayLastLogonInfo" -Type DWord -Value 1
    ```
    
- **💡 最简选择**：**GUI 法最简单**。
- **📸 验证与截图**：在 Windows 10 客户端刷新策略，注销并重新登录域用户。进入桌面之前，屏幕会弹出一个**专属交互提示框，显示“您上次成功登录此计算机的时间为...”**。直接对该提示界面进行截图。

### 步骤 4：创建组织单位“市场部”，设置即使关闭“防止容器被意外删除”也绝对无法删除

*(采用 100% 封死的全域死锁逻辑)*

- **图形界面法（GUI）**：
    1. 打开 Active Directory 用户和计算机（`dsa.msc`），点击顶部 **“查看”** → 勾选 **“高级功能”**。
    2. 右键域名 `js2025.com` 新建组织单位 **“市场部”**，把 **ADM组**、**ITD23** 和 **“云主机2”** 计算机对象移入其中。
    3. 右键点击 **“市场部”** OU → **“属性”** → **“安全”** 选项卡 → **“高级”**。
    4. 点击 **“添加”** → **“选择主体”** → 输入 `Everyone` 并确定。
    5. 将 **“类型”** 改为 **“拒绝”**；将 **“套用到”** 改为 **“这个对象及所有子对象”**（*下发继承*）。
    6. 点击“显示高级权限”，必须精准勾选三项：**“删除”**、**“删除子树”**、**“删除子对象”**。一路确定并同意安全警告。
- **命令行法（CLI）**：在主域控的命令提示符（CMD）中运行：DOS
    
    ```
    dsacls "OU=市场部,DC=js2025,DC=com" /D Everyone:SDDTDC
    ```
    
- **💡 最简选择**：**GUI 法最稳妥**。`SDDTDC` 这种底层权限缩写在考场上极易漏字导致封锁失败。
- **📸 验证与截图**：右键点击“市场部”属性，在“对象”页签中**故意取消勾选**“防止容器被意外删除”，点击应用。随后在外部右键点击“市场部”选择 **“删除”**。
**截图要点**：系统依然会强制弹窗报错：**“删除对象 市场部 失败。拒绝访问。”** 截取该“已关闭防误删保护却依旧拒绝访问”的冲突对比图。

### 步骤 5：启用 js2025.com 域的允许还原误删除对象功能（AD 回收站）

- **图形界面法（GUI）**：打开 Active Directory 管理中心（`dsac.exe`），左侧点击 `js2025(本地)`，在最右侧的“任务”面板中，点击 **“启用回收站...”**。
- **命令行法（CLI）**：在主域控上打开 PowerShell，直接运行：PowerShell
    
    ```
    Enable-ADOptionalFeature -Identity 'Recycle Bin Feature' -Scope ForestOrConfigurationSet -Target 'js2025.com'
    ```
    
- **💡 最简选择**：**命令行（CLI）法最简单**。Server 2019 的 ADAC 管理中心首次加载极慢，PowerShell 命令一键粘贴仅需 2 秒。
- **📸 验证与截图**：在 PowerShell 中输入查询命令：PowerShell
    
    ```
    Get-ADOptionalFeature -Filter "Name -eq 'Recycle Bin Feature'" | Select-Object Name, EnabledScopes
    ```
    
    **截图要点**：输出结果中 `EnabledScopes` 清晰显示了当前网域的全局配置路径，证明回收站已生效。
    

### 步骤 6：禁用 USB 可移动存储（禁止 U 盘使用）

- **图形界面法（GUI）**：在组策略编辑器中导航到：`计算机配置` → `策略` → `管理模板` → `系统` → `可移动存储访问`。双击 `所有可移动存储类: 拒绝所有权限`，设置为 **“已启用”**。
- **命令行法（CLI）**：在主域控的 PowerShell 中运行：PowerShell
    
    ```
    Set-GPRegistryValue -Name "Default Domain Policy" -Key "HKLM\Software\Policies\Microsoft\Windows\RemovableStorageDevices" -ValueName "Deny_All" -Type DWord -Value 1
    ```
    
- **💡 最简选择**：**GUI 法最简单**。
- **📸 验证与截图**：在 Windows 10 客户端上刷新策略后，插入 U 盘或尝试双击打开任意移动驱动器。
**截图要点**：系统弹出无情报错：**“位置不可用，无法访问。拒绝访问。”** 对其进行截图。

### 步骤 7：配置域密码策略：长度≥8，复杂度启用，锁定阈值5次，锁定时间15分钟

- **图形界面法（GUI）**：编辑 `Default Domain Policy`。
    1. 导航到 `账户策略` → `密码策略`：将 `密码必须满足复杂性要求` 设为 **已启用**；将 `密码长度最小值` 设为 **8**。
    2. 导航到 `账户锁定策略`：将 `账户锁定阈值` 设为 **5** 次；将 `账户锁定时间` 和 `重置账户锁定计数器` 手动修改为 **15** 分钟。
- **命令行法（CLI）**：在主域控的 PowerShell 中运行一键下发指令：PowerShell
    
    ```
    Set-ADDefaultDomainPasswordPolicy -Identity "js2025.com" -ComplexityEnabled $true -MinPasswordLength 8 -LockoutThreshold 5 -LockoutDuration "00:15:00" -LockoutObservationWindow "00:15:00"
    ```
    
- **💡 最简选择**：**命令行（CLI）法最简单**。GUI 法中各策略互相关联，修改时会频繁弹出 30 分钟系统强制同步的冲突提示，命令行可一键完美覆盖。
- **📸 验证与截图**：在 Windows 10 客户端上打开管理员权限的 CMD，运行：DOS
    
    ```
    net accounts
    ```
    
    **截图要点**：输出中清晰显示 `最小密码长度: 8`、`锁定阈值: 5`、`锁定持续时间(分): 15`。直接对该终端窗口截图。
    

### 步骤 8：限制使用 cmd，除了 Sales 组

*(全域封锁，精准豁免 Sales 组)*

- **图形界面法（GUI）**：
    1. 新建 GPO 命名为 `限制CMD` 并在域上挂载链接。
    2. 编辑该 GPO，导航到：`计算机配置` → `策略` → `Windows 设置` → `安全设置` → `软件限制策略`。右键选择 **“创建软件限制策略”**。
    3. 右键点击下方的 **“其他规则”** → **“新建路径规则”**。在路径中输入 `%SystemRoot%\system32\cmd.exe`，安全级别选 **“不允许”**，确定。
    4. 关闭编辑器回到组策略管理器，左键选中 `限制CMD` GPO。
    5. 在右侧 **“安全筛选”** 中，**删除** `Authenticated Users`；点击添加，把 `Domain Users` 和 `Sales` 组加进来。
    6. 切换到上方的 **“委派”** 选项卡，点击右下角 **“高级”**。
    7. 选中 **Sales 组**，在下方权限列表中找到 **“应用组策略”**，勾选其后面的 **“拒绝”**。点击确定。
- **命令行法（CLI）**：由于涉及修改 Active Directory 安全描述符（ACL）和底层二进制策略文件的封装，命令行极为冗长繁琐，不推荐在考场使用。
- **💡 最简选择**：**GUI 法最简单**。
- **📸 验证与截图（需要提供两张截图形成强烈对比）**：
    - **第一张截图（非 Sales 用户）**：用普通域用户登录 Win10，运行 `cmd`。
    **截图要点**：弹出报错：*“由于软件限制策略，此程序已被阻止...”*
    - **第二张截图（Sales 用户）**：切换到 Sales 组的用户登录 Win10，刷新策略后运行 `cmd`。
    **截图要点**：**CMD 黑色控制台窗口被完美、成功地开启**。

### 步骤 9：设置强制性用户配置文件（Mandatory User Profile）

*(让特定用户的任何桌面修改在注销后灰飞烟灭)*

- **图形界面法（GUI）**：
    1. **创建共享空间**：在主域控上新建文件夹 `C:\Profiles`，将其共享，共享名设为 `Profiles$`，赋予 `Everyone` 读取权限。
    2. **制作模板**：在 Windows 10 上用一个普通测试域用户登录，把桌面壁纸、快捷方式等布置好。注销该用户。
    3. **抓取配置文件**：在 Win10 上改用 **域管理员** 登录。按 `Win + R` 输入 `sysdm.cpl` 打开系统属性 → **“高级”** → 用户配置文件点击 **“设置”**。选中刚才那个测试用户的配置文件，点击 **“复制到”**，路径填入：`\\Server01\Profiles$\mandatory.v6`（*注：Win10 1809 及以上系统后缀必须加 `.v6`*）。在“允许使用”中添加 `Everyone`。
    4. **开启强制属性**：在域控上打开该共享文件夹下的 `mandatory.v6` 目录，显示隐藏文件，将里面的 **`ntuser.dat`** 重命名为 **`ntuser.man`**（`*.man` 即代表 Mandatory 强制*）。
    5. **绑定目标用户**：在 `dsa.msc` 中双击需要执行强制配置的用户，切换到 **“配置文件”** 选项卡。在 **“配置文件路径”** 中输入：`\\Server01\Profiles$\mandatory`（*切记：这里不要写 `.v6`，系统会自动识别*）。
- **命令行法（CLI）**：在完成文件重命名后，绑定用户路径可以使用 PowerShell 一键完成：PowerShell
    
    ```
    Set-ADUser -Identity "目标用户名" -ProfilePath "\\Server01\Profiles$\mandatory"
    ```
    
- **💡 最简选择**：文件复制和重命名采用 **GUI** 最直观；最后的用户路径绑定采用 **CLI（PowerShell）** 效率最高。
- **📸 验证与截图**：
    1. 用被绑定的目标用户登录 Windows 10 客户端。
    2. 在桌面上**故意新建**一个名为 `Test_Folder` 的文件夹。
    3. 点击开始菜单，选择 **“注销”** 该用户。
    4. 重新登录该用户。
    **截图要点**：进入桌面后，刚才新建的 `Test_Folder` **已经彻底消失，桌面恢复如初**。此时在客户端打开管理员运行 `sysdm.cpl` -> 高级 -> 用户配置文件设置，截图显示该用户的配置文件类型已变成了 **“强制” (Mandatory)**。
