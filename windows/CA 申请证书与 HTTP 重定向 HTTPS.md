# CA 申请证书与 HTTP 重定向 HTTPS

**环境背景：**

- **操作系统：** Windows Server (装有 IIS)
- **证书颁发机构：** 内部企业 CA (例如 `skills-SUN-CA`)
- **目标域名：** `web.skills.com`
- **站点架构：**
    - 站点 A (`bbb`)：作为入口，绑定 HTTP (端口 80)
    - 站点 B (`web`)：作为终点，绑定 HTTPS (端口 443)

---

## 阶段一：向企业 CA 申请并颁发服务器证书

此阶段的目标是获取一张受内网信任的 SSL 证书，并将其存放在服务器的正确位置。

### 步骤（通过 MMC 控制台）：

1. **打开控制台：** 按下 `Win + R` 键，输入 `mmc` 并回车。
2. **添加证书管理单元：**
    - 点击左上角 **文件 (File)** -> **添加/删除管理单元 (Add/Remove Snap-in)**。
    - 在左侧列表中选择 **证书 (Certificates)**，点击 **添加 (Add)**。
    - 选择 **计算机帐户 (Computer account)** -> 下一步 -> 选择 **本地计算机 (Local computer)** -> 完成。点击 **确定**。
3. **开始申请证书：**
    - 在左侧导航栏展开：**证书 (本地计算机) -> 个人 -> 证书**。
    - 在中间空白处右键点击，选择 **所有任务 (All Tasks)** -> **申请新证书 (Request New Certificate)**。
4. **填写证书信息：**
    - 一路点击“下一步”，直到出现证书模板列表（通常选择 `Web 服务器` 或管理员配置好的模板）。
    - 点击模板下方提示的 **“需要更多信息去注册此证书”** 的蓝色链接。
    - **使用者 (Subject) 选项卡：** 类型选择 **公用名 (Common Name)**，值为 `.skills.com`（或具体的 `web.skills.com`），点击 **添加**。
    - **常规 (General) 选项卡：** 填写友好名称（如：`skills-wildcard-cert`），方便后续在 IIS 中识别。
    - 点击 **确定**，然后勾选该模板，点击 **注册 (Enroll)**。
5. **验证：** 打开 IIS 管理器，点击左侧服务器根节点，双击右侧的 **“服务器证书”**，确认新申请的证书已在列表中。

---

## 阶段二：配置 HTTPS 目标站点 (站点 "web")

此阶段的目标是将申请好的证书绑定到真正的网站上，并**确保它自身不会发生重定向死循环**。

### 步骤：

1. **绑定 HTTPS 与证书：**
    - 在 IIS 管理器左侧，展开网站列表，点击目标站点 **`web`**。
    - 在右侧操作栏点击 **绑定... (Bindings...)**。
    - 点击 **添加 (Add)**：
        - **类型：** 选择 `https`
        - **端口：** 保持 `443`
        - **主机名：** 填写 `web.skills.com` (若使用 SNI 则需勾选需要服务器名称指示)
        - **SSL 证书：** 在下拉菜单中选择刚刚申请的那个证书 (例如 `skills-wildcard-cert`)。
    - 点击 **确定** 并关闭。
2. **⚠️ 关键避坑：关闭自身重定向**
    - 依然选中站点 **`web`**。
    - 在中间功能视图中，双击打开 **HTTP 重定向 (HTTP Redirect)**。
    - **务必取消勾选** “将请求重定向到此目标”。
    - 如果做了修改，点击右侧的 **应用 (Apply)**。

---

## 阶段三：配置 HTTP 入口站点实现自动跳转 (站点 "bbb")

此阶段的目标是拦截所有访问 80 端口（HTTP）的用户，并将他们强制引导至 443 端口（HTTPS）。

### 步骤：

1. **确认基础绑定：**
    - 点击站点 **`bbb`**。确认其绑定只有 `http`，端口 `80`。
2. **设置重定向规则：**
    - 选中站点 **`bbb`**，双击打开中间的 **HTTP 重定向 (HTTP Redirect)**。
    - **勾选** “将请求重定向到此目标”。
    - 在目标地址框中填写完整的安全 URL：`https://web.skills.com`
3. **配置重定向行为：**
    - 勾选 **“仅将请求重定向到此目录(非子目录)中的内容”**（按需勾选，通常用于精确跳转）。
    - 状态代码 (Status code) 选择：**永久 (301)**。*(注：301 有利于 SEO，并让浏览器记住该跳转)*。
4. **保存配置：**
    - 点击右侧操作栏的 **应用 (Apply)**。

---

## 阶段四：测试与验证

1. **清理环境：** 由于浏览器会强烈缓存 301 重定向记录，测试前**必须**打开浏览器的 **无痕/隐私模式 (Incognito Mode)**。
2. **执行测试：** 在地址栏输入 `http://web.skills.com` 并回车。
3. **预期结果：**
    - 浏览器地址栏应瞬间自动变更为 `https://web.skills.com`。
    - 地址栏左侧应出现一把**安全小锁**（因为企业 CA 根证书在域环境中是默认受信任的）。
    - 页面正常显示网站内容（如：`hello, windows`）。
