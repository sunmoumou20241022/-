## 任务一：域环境搭建

### 步骤1：配置 dc1 的 IP 网络（题目要求用命令）

```powershell
# 先确认网卡实际名称（英文版可能不是"Ethernet"）
Get-NetAdapter | Select Name, InterfaceDescription

# 设置静态IP（网卡名根据上面结果修改）
New-NetIPAddress -InterfaceAlias "Ethernet0" -IPAddress 192.168.10.30 -PrefixLength 24 -DefaultGateway 192.168.10.254

# DNS指向自己（dc1要做DNS服务器）
Set-DnsClientServerAddress -InterfaceAlias "Ethernet0" -ServerAddresses 127.0.0.1
```

> ⚠️ **陷阱2**：英文版网卡名可能是 `Ethernet0`、`Ethernet 2` 等，不是中文版的"以太网"。必须先用 `Get-NetAdapter` 确认。

### 步骤2：搭建域服务（题目要求用命令）

```powershell
# 安装 AD DS 角色
Install-WindowsFeature AD-Domain-Services -IncludeManagementTools

# 创建新林根域
$secPassword = ConvertTo-SecureString "Skills39!" -AsPlainText -Force
Install-ADDSForest -DomainName "skills2026.com" -SafeModeAdministratorPassword $secPassword -Force
```

> 安装后自动重启。重启后用 `SKILLS2026\Administrator` 登录，密码 `Skills39!`。

### 步骤3：将 dc2 和 client 加入域（题目要求用命令）

**在 dc2 上：**

```powershell

New-NetIPAddress -InterfaceAlias "Ethernet0" -IPAddress 192.168.10.40 -PrefixLength 24 -DefaultGateway 192.168.10.254
Set-DnsClientServerAddress -InterfaceAlias "Ethernet0" -ServerAddresses 192.168.10.30
```

### 步骤5：创建组织单位和用户（没要求命令，但命令最简便）

> ⚠️ **陷阱4**：IT 是**全局安全组（Global Security Group）**，HR 是**本地安全组（Domain Local Security Group）**！
> ⚠️ **陷阱5**：IT 用户名 `IT1`~`IT30`（无前导零），HR 用户名 `HR01`~`HR30`（有前导零）！

在 dc1 上：

```powershell
# ========== IT 组织单位 ==========
# 创建OU
New-ADOrganizationalUnit -Name "IT" -Path "DC=skills2026,DC=com"

# 创建全局安全组
New-ADGroup -Name "IT" -SamAccountName "IT" -GroupCategory Security -GroupScope Global -Path "OU=IT,DC=skills2026,DC=com"

# 批量创建 IT1-IT30（无前导零）
$secPassword = ConvertTo-SecureString "Skills39!" -AsPlainText -Force
1..30 | ForEach-Object {
    $name = "IT$_"
    New-ADUser -Name $name -SamAccountName $name -UserPrincipalName "$name@skills2026.com" `
        -Path "OU=IT,DC=skills2026,DC=com" -AccountPassword $secPassword -Enabled $true -CannotChangePassword $true
    Add-ADGroupMember -Identity "IT" -Members $name
}

# ========== HR 组织单位 ==========
# 创建OU
New-ADOrganizationalUnit -Name "HR" -Path "DC=skills2026,DC=com"

# 创建本地安全组（DomainLocal 不是 Global！）
New-ADGroup -Name "HR" -SamAccountName "HR" -GroupCategory Security -GroupScope DomainLocal -Path "OU=HR,DC=skills2026,DC=com"

# 批量创建 HR01-HR30（有前导零）
1..30 | ForEach-Object {
    $num = "{0:D2}" -f $_
    $name = "HR$num"
    New-ADUser -Name $name -SamAccountName $name -UserPrincipalName "$name@skills2026.com" `
        -Path "OU=HR,DC=skills2026,DC=com" -AccountPassword $secPassword -Enabled $true -CannotChangePassword $true
    Add-ADGroupMember -Identity "HR" -Members $name
}
```

> `-CannotChangePassword $true` = 禁止用户修改口令，题目要求。

### 步骤6：验证用户创建（题目指定命令）

## 任务二：证书和DNS服务

### 步骤7：配置 DNS 正反向解析（没要求命令，但命令最简便）

在 dc1 上：

```powershell
# 创建反向查找区域
Add-DnsServerPrimaryZone -NetworkId "192.168.10.0/24" -ReplicationScope Forest

# 创建主机记录（dc1自己的记录在加域时已自动创建）
Add-DnsServerResourceRecordA -Name "dc2" -ZoneName "skills2026.com" -IPv4Address 192.168.10.40 -CreatePtr
Add-DnsServerResourceRecordA -Name "client" -ZoneName "skills2026.com" -IPv4Address 192.168.10.50 -CreatePtr
```

### 步骤8：配置未知域名解析到 dc1

题目要求：DNS 服务器允许解析任何未知域名的任何未知记录，将其解析为 dc1 的 IP 地址。

```powershell
# 创建根区域 "." —— 使dc1成为互联网根DNS服务器
Add-DnsServerPrimaryZone -Name "." -ZoneFile "root.dns"

# 在根区域创建通配符A记录
Add-DnsServerResourceRecordA -Name "*" -ZoneName "." -IPv4Address 192.168.10.30
```

> 创建根区域后，dc1 认为自己是互联网的根，所有未知域名都会在根区域查找，通配符 `*` 确保任何域名都解析到 dc1。

验证（题目指定命令）：

```powershell
# (1) 检查正向DNS区域
nslookup - 192.168.10.30
# 进入交互界面后输入：ls skills2026.com

# (2) 检查未知域名解析（随机域名应解析到192.168.10.30）
nslookup "$((1..8 | ForEach-Object {[char](Get-Random -Minimum 97 -Maximum 123)}) -join '').$(@('com','net','org','top') | Get-Random)"
```

> ⚠️ **陷阱6**：需要在 DNS 管理器中右键服务器 → **Properties** → **Root Hints** 选项卡，删除所有根提示（否则未知域名可能先查根提示而不是本地区域）。或者确保根区域 "." 创建成功后，根提示会自动失效。

### 步骤9：批量创建 skills1.cn ~ skills100.cn 区域（没要求命令，但命令最简便）

```powershell
1..100 | ForEach-Object {
    $zone = "skills$_.cn"
    Add-DnsServerPrimaryZone -Name $zone -ZoneFile "$zone.dns"
    Add-DnsServerResourceRecordA -ZoneName $zone -Name "@" -IPv4Address 192.168.10.30
}
```

验证：

1. DNS Manager → **View** → **List** 模式 → 截图应包含 skills1.cn ~ skills100.cn 共100个区域
2. `nslookup skills31.cn` → 应解析到 192.168.10.30

### 步骤10：安装证书服务（企业根CA）（没要求命令，命令最简便）

```powershell
# 安装AD CS角色
Install-WindowsFeature Adcs-Cert-Authority -IncludeManagementTools

# 配置为企业根CA
Install-AdcsCertificationAuthority -CAType EnterpriseRootCA -CACommonName "Skills-RootCA" -ValidityPeriod Years -ValidityPeriodUnits 2 -Force
```

> ⚠️ **陷阱7**：必须是 **EnterpriseRootCA**（企业根CA），不是 Standalone。只有企业CA才能自动响应域用户证书申请、使用证书模板。

验证（题目指定命令）：

```powershell
certutil -cainfo
certutil -TCAInfo | Select-String "Years"
```

应显示 CA 名称为 `Skills-RootCA`，有效期为 `2 Years`。截图。

### 步骤11：申请 Web 服务器证书

> ⚠️ **陷阱8**：证书模板操作涉及 AD Schema 修改，全命令行极易出错，**推荐 GUI**。

## 任务三：IIS服务

### 步骤12：创建 HTTPS 安全静态网站

**步骤12a：安装 IIS（在 dc2 上，命令最简便）**

```powershell
Install-WindowsFeature -Name Web-Server -IncludeManagementTools
```

**步骤12b：创建网站目录和首页（命令最简便）**

```powershell
# 创建目录
New-Item -Path "C:\Web_Files\Web_Root" -ItemType Directory -Force

# 创建首页
Set-Content -Path "C:\Web_Files\Web_Root\index.html" -Value "<html><body><h1>技能成才，技能报国!</h1></body></html>" -Encoding UTF8
```

**步骤12c：配置网站绑定（GUI最可靠）**

1. 打开 IIS Manager（`inetmgr`）
2. 右键 **Sites** → **Add Website**
   - Site name: `skills_web`
   - Physical path: `C:\Web_Files\Web_Root`
   - Type: `https`
   - Port: `443`
   - Host name: `www.skills2026.com`
   - SSL certificate: 选择 `scrt`
3. 点击 **OK**
4. 删除默认的 Default Web Site（或停止它），避免端口冲突

**步骤12d：确保 DNS 解析 www.skills2026.com**

在 dc1 上（如果还没创建 www 的 A 记录）：

```powershell
Add-DnsServerResourceRecordA -Name "www" -ZoneName "skills2026.com" -IPv4Address 192.168.10.40 -CreatePtr
```

**步骤12e：在 client 上测试**

1. 确保 client 的 DNS 指向 dc1（192.168.10.30）
2. 在 client 上运行 `gpupdate /force` 强制刷新组策略，确保收到根CA证书

4. 应显示"技能成才，技能报国!"，**不出现证书警告**

### 步骤13：建立 FTP 站点

**步骤13a：安装 FTP 服务（在 dc2 上，命令最简便）**

```powershell
Install-WindowsFeature -Name Web-Ftp-Server -IncludeManagementTools
```

**步骤13b：创建目录结构（命令最简便）**

> ⚠️ **陷阱13**：FTP 用户隔离的目录结构是关键！域用户的目录必须在 `C:\ftp\域名\用户名\` 下，匿名用户的目录在 `C:\ftp\LocalUser\Public\` 下。

```powershell
# 创建FTP根目录
New-Item -Path "C:\ftp" -ItemType Directory -Force

# 创建域用户隔离目录（域名用NetBIOS名 skills2026）
New-Item -Path "C:\ftp\skills2026\IT1" -ItemType Directory -Force
New-Item -Path "C:\ftp\skills2026\IT2" -ItemType Directory -Force

# 创建匿名用户目录
New-Item -Path "C:\ftp\LocalUser\Public" -ItemType Directory -Force

# 创建测试文件
Set-Content -Path "C:\ftp\skills2026\IT1\IT1.txt" -Value "IT1 test file"
Set-Content -Path "C:\ftp\skills2026\IT2\IT2.txt" -Value "IT2 test file"
```

> ⚠️ **陷阱14**：域名文件夹名必须是域的 **NetBIOS 名称**，不是完整域名。`skills2026.com` 的 NetBIOS 名是 `SKILLS2026`（创建域时自动生成），在 FTP 隔离中用 `skills2026`（不区分大小写）。可以通过在 dc1 上运行 `nbtstat -n` 或查看 AD UC 中的域名属性来确认。

**步骤13c：创建 FTP 站点（GUI最简便）**

1. 打开 IIS Manager（`inetmgr`）
2. 右键 **Sites** → **Add FTP Site**
   - FTP site name: `skills_ftp`
   - Physical path: `C:\ftp`
   - Next
3. Binding and SSL：
   - IP Address: 选择 `192.168.10.40`
   - Port: `21`
   - SSL: 选择 **Allow SSL**（或不要求SSL）
   - Next
4. Authentication and Authorization：
   - Authentication: 勾选 **Anonymous** + **Basic**
   - Allow access: **All users**（或 Specified users → `IT1, IT2`）
   - Permissions: 勾选 **Read** + **Write**
   - Finish

**步骤13d：配置 FTP 用户隔离**

1. 在 IIS Manager 中点击 FTP 站点
2. 双击 **FTP User Isolation**
3. 选择 **User name directory (disable global virtual directories)**
4. 右侧点击 **Apply**

**步骤13e：配置 FTP Banner 消息**

1. 点击 FTP 站点
2. 双击 **FTP Messages**
3. 在 **Banner** 文本框中输入：`欢迎访问 FTP服务器`
4. 右侧点击 **Apply**

**步骤13f：关闭 dc2 防火墙（测试用）**

```powershell
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False
```

验证（题目指定命令）：

```powershell
# (1) 在dc2上检查FTP目录结构
tree C:\ftp /f

# (2) 在dc2上检查Banner消息
ftp 192.168.10.40
# 登录前应看到 "220-欢迎访问 FTP服务器"

# (3) 在client上测试FTP隔离
# 浏览器打开 ftp://192.168.10.40
# 用 skills2026\IT2 登录，密码 Skills39!
# 应只看到 IT2 自己的目录（IT2.txt）
```

> ⚠️ **陷阱15**：FTP 登录域用户时，用户名格式必须是 `skills2026\IT2` 或 `IT2@skills2026.com`（带域名前缀），否则 FTP 无法识别为域用户，会走 LocalUser 路径导致隔离失败。

## 陷阱汇总

| #   | 陷阱           | 说明                                                           |
| --- | ------------ | ------------------------------------------------------------ |
| 1   | 子网掩码 `/254`  | 题目笔误，实际应为 `/24`（255.255.255.0）                               |
| 2   | 网卡名称         | 英文版是 `Ethernet0` 不是中文"以太网"，用 `Get-NetAdapter` 确认             |
| 3   | 加域前DNS       | dc2 和 client 加域前必须将 DNS 指向 dc1（192.168.10.30）                |
| 4   | IT vs HR 组类型 | IT=全局安全组（Global），HR=本地安全组（DomainLocal）                       |
| 5   | 用户名格式        | IT1~IT30 无前导零，HR01~HR30 有前导零                                 |
| 6   | DNS根提示       | 创建根区域 "." 后，检查根提示是否自动失效，否则未知域名可能走根提示                         |
| 7   | 企业CA         | 必须选 EnterpriseRootCA，不是 Standalone，否则无法使用模板/自动响应             |
| 8   | 证书模板         | 必须先 Duplicate Template 再修改，不能直接用 Web Server 原模板              |
| 9   | 模板名称         | 英文版中模板名显示为 "Web Server Copy"，注册时以此为准                         |
| 10  | 证书SAN        | 必须包含 `*.skills2026.com` 和 `skills2026.com`，否则浏览器报证书名称不匹配     |
| 11  | Add 按钮       | 每填一项证书属性都要点 Add，否则不生效                                        |
| 12  | 根CA信任        | client 必须 `gpupdate /force` 刷新组策略才能收到根CA证书                   |
| 13  | FTP目录结构      | 域用户：`C:\ftp\skills2026\用户名\`，匿名用户：`C:\ftp\LocalUser\Public\` |
| 14  | NetBIOS名     | FTP 隔离目录中的域名文件夹必须用 NetBIOS 名（`skills2026`），不是完整域名            |
| 15  | FTP登录格式      | 域用户登录FTP时用户名必须带域名前缀：`skills2026\IT2`                         |
