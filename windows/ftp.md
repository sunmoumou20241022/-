# ftp

**安装FTP服务**

- 在作为FTP服务器的虚拟机（在第二台虚拟机上），打开服务器管理器。
- 点击“添加角色和功能”，进入向导，安装类型选择“**基于角色或基于功能的安装**”。
- 在“服务器角色”界面，勾选“**Web服务器(IIS)**”。
- 在弹出的窗口中，需要确保勾选了“**FTP服务器**”及“**FTP扩展性**”。
- 之后一路默认，点击“安装”即可。

**配置防火墙（第二台）**

操作→属性→把前三个配置文件都关上

**创建FTP用户**

要实现用户隔离，需要先有对应的系统用户。（在第一台虚拟机上）

- 在服务器管理器中，点击左上角的“工具”，选择“**计算机管理**”。
- 展开“本地用户和组” -> “用户”。在右侧空白处右键，选择“**新用户**”。
- 分别创建zhangsan和 lisi 两个本地用户，并设置符合要求的密码。

**规划并创建FTP目录结构（第二台）**

文件夹的**命名和位置**是实现用户隔离的关键。

- 在FTP服务器的任意位置（例如C:\盘下），创建一个根目录，比如命名为FTPSite。
- 根据你的需求，在这个根目录下创建如下文件夹结构：

text

C:\FTP

│

├─localuser\

│    ├─public\ (对应匿名用户)

│

│

└─skills.com\ (NetBIOS域名)

├─public\     (用于存放公共文件)

├─zhangsan\   (对应域用户)

└─lisi\       (对应域用户)

- localuser文件夹是固定要求。
- com是域控制器 (DC) 的**NetBIOS域名**。如果域控制器完整域名是skills.com，那么NetBIOS域名就是 SKILLS 或 skills.com，你需要通过 nbtstat -n 命令或在AD用户与计算机的属性中确认。
- 在各自对应的文件夹（zhangsan，lisi）里，放置一些不同的测试文件，方便后续验证。

**配置FTP站点 (启用用户隔离与域名访问)**

现在可以正式配置你的FTP站点。

- 打开**IIS管理器**。
- 在左侧“连接”窗格中，右键点击“网站”，选择“**添加FTP站点...**”。
- **站点信息**：
    - **FTP站点名称**：填一个易识别的名字，如Skills_FTP。
    - **物理路径**：选择我们上一步创建的根目录，即C:\FTP。
- **绑定和SSL设置**：
    - **IP地址**：输入本机的ip地址。
    - **端口**：保持默认的21。
    - **SSL**：选择“**无SSL**”。
    - 保存后，你就可以用这个ip访问FTP了。
- **身份验证和授权信息**：
    - **身份验证**：勾选“**基本**”。如果允许匿名访问，可以同时勾选“匿名”。
    - **授权**：
        - **允许访问**：选择“指定用户”，然后填入zhangsan, lisi。
        - **权限**：根据需要为所有用户勾选“**读取**”和“**写入**”。
    - 点击“完成”后，回到IIS管理器，选中你刚创建好的FTP站点。
    - 在右侧的“管理”区域，点击“**FTP用户隔离**”。
    - 选择“**用户名目录（禁用全局虚拟目录）**”，然后点击右侧的“应用”。至此，用户隔离功能配置完毕。

**Win10客户端访问测试**

在另一台Win10客户机上，可以用以下任一方式验证：

- **文件资源管理器**：直接输入[ftp://192.168.xx.xx](ftp://192.168.xx.xx/)或 [ftp://ftp.skills.com](ftp://ftp.skills.com/)，并按提示输入用户名和密码。（一般都是用ip地址，如果用域名访问的话还要从新设置）
- **命令行**：打开CMD，输入ftp 192.168.xx.xx或 ftp ftp.skills.com，然后根据提示输入用户名和密码。
- **登录用户名格式**：
    - 访问本地用户（localuser下）目录时，直接输入用户名zhangsan或 lisi。
    - 访问域用户（com下）目录时，必须输入**完整格式**ftp.skills.com|zhangsan。
