# 跨交换机非对称安全隔离（魔术链路）

**1. 最终实现的通信矩阵（完美通关状态）**
在**完全不改变**四台 PC 的物理 VLAN 归属（PC1/3 属 VLAN 10，PC2/4 属 VLAN 20）且处于**同一网段**（`192.168.100.x`）的前提下，我们成功实现了以下看似矛盾的二层安全矩阵：**发起端目的端业务状态预期现象底层隔离机制PC1** (V10)**PC4** (V20)**特权互通**✅ 成功 (Reply)互联 Hybrid 链路进行 **PVID 错配**，在线路上将标签“偷梁换柱”。**PC1** (V10)**PC3** (V10)**同VLAN隔离**❌ 超时 (Timeout)流量跨越网线后被强行改写为 VLAN 20，无法进入 PC3 的 VLAN 10 接口。**PC1** (V10)**PC2** (V20)**本地隔离**❌ 无法访问本地交换机内部 Access 接口天然的二层物理壁垒，广播域不相交。
**2. 核心底层破局点：Hybrid 错配（PVID Mismatch）**
传统的 Trunk 骨干链路要求两端配置一模一样，而华为特有的 **Hybrid（混杂）模式** 允许我们打破常规，在中间的物理电缆上玩一场“标签魔术”：
• **SW1 的 G0/0/3 配置**：`pvid vlan 10` 且 `untagged vlan 10`
• **SW2 的 G0/0/3 配置**：`pvid vlan 20` 且 `untagged vlan 20`
**3. 三大路径的微观生死劫（数据包变身记）
🟢 为什么 PC1 能 Ping 通跨 VLAN 的 PC4？**
1. **出 SW1**：PC1 发出的无标签包进入 SW1，被 G0/0/1 (Access 10) 打上 **`Tag 10`**。走到互联口 G0/0/3 时，由于标签（10）等于该接口的 PVID（10），标签被**无情剥离**，变成**纯净无标签包**赤裸裸地流过网线。
2. **入 SW2**：无标签包抵达 SW2 的 G0/0/3，接口发现是无标签包，立刻无脑注入自己的 PVID——**`Tag 20`**！
3. **触达目标**：此时数据包在 SW2 内部变成了正统的 VLAN 20 流量，顺利找到连在 Access 20 上的 PC4。
4. *回包过程同理，流量在回程通过网线时，又从小路被改写回了 VLAN 10，完美闭环！*
**🔴 为什么 PC1 绝对 Ping 不通同 VLAN 的 PC3？**
• PC1 的包通过魔术链路到达 SW2 内部时，逻辑身份已经变成了 **`Tag 20`**。
• 当 SW2 试图把包发给 PC3 时，发现 PC3 所在的 G0/0/1 接口是个死板的 `Access vlan 10`。
• **结局**：`Tag 20` 试图强闯 `VLAN 10` 的大门，被接口保安在硬件芯片层面**直接丢弃 (Drop)**。
**🔴 为什么 PC1 绝对 Ping 不通同交换机的 PC2？**
• 流量根本没有走中间的魔术网线。PC1 的包在 SW1 内部带着 **`Tag 10`** 标签，而本地的 PC2 缩在 `Access vlan 20` 的港湾里。
• **结局**：在同台交换机内部，不同 VLAN 的数据犹如平行宇宙，连 ARP 请求都收不到。
**💡 实验通关核心口诀（应付老师提问神技）**“**入向看 PVID 打标签，出向看 untagged 列表脱衣服。** 常规 Trunk 链路两端必须对称，而本实验利用 Hybrid 接口在物理中继链路上进行了**非对称错配**。数据包在网线上以 **Untagged** 形态传输，从而在对端落户时被‘强行改写’了 VLAN 身份，完美实现了跨域穿透与同域隔离的奇特效果。”

### SW1 (左侧交换机) 终极全量代码：

Plaintext

```
<Huawei> system-view
[Huawei] sysname SW1

# 1. 全局创建部门
[SW1] vlan batch 10 20

# 2. 配置 PC1 接入接口 (VLAN 10)
[SW1] interface GigabitEthernet 0/0/1
[SW1-GigabitEthernet0/0/1] port link-type access
[SW1-GigabitEthernet0/0/1] port default vlan 10
[SW1-GigabitEthernet0/0/1] quit

# 3. 配置 PC2 接入接口 (VLAN 20)
[SW1] interface GigabitEthernet 0/0/2
[SW1-GigabitEthernet0/0/2] port link-type access
[SW1-GigabitEthernet0/0/2] port default vlan 20
[SW1-GigabitEthernet0/0/2] quit

# 4. 配置中间的【魔术中继接口】
[SW1] interface GigabitEthernet 0/0/3
[SW1-GigabitEthernet0/0/3] port link-type hybrid
[SW1-GigabitEthernet0/0/3] port hybrid pvid vlan 10
[SW1-GigabitEthernet0/0/3] port hybrid untagged vlan 10
[SW1-GigabitEthernet0/0/3] quit
```
