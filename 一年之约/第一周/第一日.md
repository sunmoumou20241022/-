## 一、Token 边界原理

### 1.1 什么是 Token？

**Token = LLM 看到的"最小单位"，不是人类看到的"字符"或"单词"。**

举个例子：句子 `"Hello world"`，人类看到的是 2 个单词，但 LLM 可能看到：

| Tokenizer 类型 | Token 切分 | Token 数量 |
|---------------|-----------|-----------|
| GPT-2 BPE | `Hello` + ` world` | 2 |
| GPT-4 BPE | `Hello` + ` world` | 2 |
| LLaMA | `Hello` + ` world` | 2 |
| 中文场景 | `你` + `好` + `世` + `界` | 4（字级） |

**关键洞察**：同一个词，在不同 tokenizer 下可能被切成不同的 token。这导致了**Token 边界问题**。

### 1.2 Token 边界问题

**定义**：Tokenizer 的切分方式和人类理解的"词边界"不一致，导致：

1. **人类看到的"一个词" ≠ LLM 看到的"一个 token"**
2. **攻击者可以通过操纵 token 边界来隐藏恶意内容**

#### 案例 1：隐藏的恶意词

```
人类看到：  "制作炸弹"
LLM 看到：  "制作" + "炸" + "弹" （3 个 token）

攻击者加空格：
"制 作 炸 弹"
→ LLM 看到：4 个独立 token，可能绕过关键词过滤
```

#### 案例 2：Unicode 操纵

```
正常文本：  "password"
攻击版本：  "passw‌ord" （中间插入零宽字符 U+200C）
人类看到：  还是 "password"
LLM 看到：  "passw" + "‌" + "ord" → 3 个 token，可能绕过检测
```

#### 案例 3：BPE 切分的意外

BPE（Byte Pair Encoding）是基于统计的切分：
- 训练数据中高频出现的字符组合会被合并成一个 token
- 低频组合会被拆开

```
"Explosive" → 可能切分为 "Explos" + "ive" 或 "Explo" + "sive"
取决于该词在训练数据中的频率
```

这意味着：**同一个词，在同一个 tokenizer 下，切分方式也可能因为上下文而变化**。

### 1.3 Token 边界攻击的意义

**为什么这很重要？**

| 传统安全检测 | Token 边界攻击后 |
|-------------|-----------------|
| 关键词匹配 `"bomb"` | `"b" + "omb"` 绕过 |
| 正则表达式 `"password"` | `"pass‌word"` 绕过 |
| 字符串过滤 `"admin"` | `"adm" + "in"` 绕过 |

**核心问题**：安全检测在"字符层"做，但 LLM 在"Token 层"理解。两层之间的映射不一致 → 攻击窗口。

### 1.4 学术研究证据

根据 arxiv 的研究：

- **Character-level vs Token-level attacks（arxiv:2405.04346）**：Token-level 攻击容易改变语义，Character-level 攻击更容易保持语义但被认为更容易防御——研究者挑战了这个假设，证明 character-level 方法对 BERT 和 Llama 2 都有效。
- **Variable-length attacks（arxiv:2104.08139）**：通过引入空白 token 进行插入、删除、替换操作，可以探索决策边界，而不局限于固定长度的对抗样本。

### 1.5 Token 边界的防御思考

1. **多层检测**：字符层 + Token 层 + 语义层都要检测
2. **Tokenizer-aware filtering**：过滤时要考虑目标模型使用的 tokenizer
3. **统计异常检测**：零宽字符、异常空格、罕见 Unicode 都是信号

## 二、Transformer 架构

> 基于 A1 论文：*Attention Is All You Need*（Vaswani et al., 2017）
> arxiv:1706.03762

### 2.1 背景：为什么 Transformer 重要？

在 Transformer 之前，序列建模的主流是：

| 模型类型 | 代表 | 问题 |
|---------|------|------|
| RNN | LSTM, GRU | 串行计算，无法并行；长序列梯度消失 |
| CNN | ConvSeq2Seq | 需要堆叠多层才能覆盖长距离依赖 |
| RNN + Attention | Bahdanau Attention | Attention 只在顶层，底层还是 RNN |

Transformer 的突破：**完全抛弃 RNN/CNN，只用 Attention**。

### 2.2 整体架构：Encoder-Decoder

```
输入序列 → [Encoder Stack] → 编码表示
                        ↓
                    [Decoder Stack] → 输出序列
```

#### Encoder（编码器）

- **N = 6 层**，每层结构相同
- 每层包含 **2 个子层**：
  1. Multi-Head Self-Attention（多头自注意力）
  2. Position-wise Feed-Forward Network（位置级前馈网络）
- 每个子层外包裹 **残差连接 + Layer Norm**
- 所有子层输出维度：**d_model = 512**

#### Decoder（解码器）

- **N = 6 层**，每层结构相同
- 每层包含 **3 个子层**：
  1. Masked Multi-Head Self-Attention（带掩码的多头自注意力）
  2. Multi-Head Attention over Encoder Output（对编码器输出做注意力）
  3. Position-wise Feed-Forward Network
- **Masked** 的作用：防止解码时"看到未来"，保持自回归特性

### 2.3 核心：Scaled Dot-Product Attention

**公式**：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**解释**：

| 符号 | 含义 |
|------|------|
| Q (Query) | "我要找什么" |
| K (Key) | "别人有什么" |
| V (Value) | "别人的内容" |
| $QK^T$ | Query 和 Key 的相似度矩阵 |
| $\sqrt{d_k}$ | 缩放因子，防止点积过大导致 softmax 梯度消失 |

**直觉**：每个位置"问"其他位置"你跟我有多相关"，然后按相关性加权取值。

### 2.4 Multi-Head Attention

**为什么需要多头？**

单头注意力只能学习一种"相关性模式"。多头让模型同时关注多种关系：
- 头 1：关注语法关系
- 头 2：关注语义关系
- 头 3：关注位置关系
- ...

**公式**：

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$

其中：
$$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

**参数**：
- h = 8 个头
- 每个头的维度：d_k = d_v = d_model/h = 64
- 每个头有独立的投影矩阵 W^Q, W^K, W^V

### 2.5 Positional Encoding

**问题**：Transformer 没有 RNN 的顺序处理，如何知道位置？

**解决方案**：在输入 embedding 上叠加位置编码。

**公式（Sinusoidal）**：

$$PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{\text{model}}})$$

$$PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{\text{model}}})$$

**特点**：
- 不同位置有不同的 sin/cos 组合
- 波长从 2π 到 10000·2π，形成几何级数
- **关键性质**：PE_{pos+k} 可以表示为 PE_pos 的线性函数 → 模型可以学习"相对位置"

### 2.6 Position-wise Feed-Forward Network

**公式**：

$$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$

**结构**：
- 两层线性变换，中间是 ReLU
- 输入输出维度：d_model = 512
- 中间维度：d_ff = 2048（扩大 4 倍）

**理解**：在每个位置独立做同样的"特征变换"，相当于对每个 token 做一次 MLP。

### 2.7 训练细节

| 项目 | Base Model | Big Model |
|------|-----------|-----------|
| d_model | 512 | 1024 |
| d_ff | 2048 | 4096 |
| h (heads) | 8 | 16 |
| N (layers) | 6 | 6 |
| Dropout | 0.1 | 0.3 |
| 参数量 | ~65M | ~213M |

**优化器**：Adam（β_1=0.9, β_2=0.98, ε=10^-9）

**学习率调度**：
$$lrate = d_{\text{model}}^{-0.5} \cdot \min(step\_num^{-0.5}, step\_num \cdot warmup\_steps^{-1.5})$$

- warmup_steps = 4000（前 4000 步线性增加）
- 之后按步数平方根衰减

**正则化**：
- Residual Dropout：每个子层输出 + embedding + pos encoding 都加 dropout
- Label Smoothing：ε_ls = 0.1（让模型更"不确定"，但提升 BLEU）

### 2.8 结果

| 任务 | BLEU | 对比 |
|------|------|------|
| WMT 2014 EN→DE | 28.4 | 比之前最佳集成高 2+ BLEU |
| WMT 2014 EN→FR | 41.8 | 单模型 SOTA |
| 英语成分句法分析 | — | 大数据和有限数据都成功 |

**训练时间**：
- Base Model：12 小时（8 GPU）
- Big Model：3.5 天（8 GPU）

**关键优势**：并行化程度高，训练时间比 RNN/CNN 模型大幅缩短。

## 三、Token 边界 + Transformer 的安全意义

### 3.1 Token 如何进入 Transformer

```
原始文本 → [Tokenizer] → Token IDs → [Embedding Layer] → 向量序列 → [Transformer Layers]
```

**关键点**：

1. **Tokenizer 是第一道门**：文本如何被切分，决定了模型"看到"什么
2. **Embedding 层是第二道门**：Token ID → 向量，决定了模型的"词汇表"
3. **Transformer 层处理向量序列**：注意力机制决定"哪些位置互相关联"

### 3.2 攻击者在哪一层操作？

| 层级 | 攻击方式 |
|------|---------|
| Tokenizer 层 | Token boundary manipulation（今日主题） |
| Embedding 层 | Embedding poisoning, adversarial tokens |
| Transformer 层 | Attention manipulation, gradient-based attacks |

## 四、关键概念速查表

### Token 边界

| 术语 | 定义 |
|------|------|
| Token | LLM 处理的最小单位，由 tokenizer 决定 |
| BPE | Byte Pair Encoding，统计驱动的子词切分 |
| Token boundary | Token 切分边界，与人类理解的词边界可能不一致 |
| Token boundary attack | 通过操纵 token 边界绕过安全检测 |

### Transformer

| 术语 | 定义 |
|------|------|
| d_model | 模型的隐藏层维度（512） |
| Multi-Head Attention | 多个并行注意力头，学习不同关系模式 |
| Self-Attention | 同一序列内部位置的相互注意力 |
| Positional Encoding | 给没有顺序感的 Transformer 加位置信息 |
| Encoder-Decoder | 编码器压缩输入，解码器生成输出 |
| Scaled Dot-Product | 点积注意力 + 缩放，防止梯度消失 |

## 六、今日能说的话

> LLM 的安全检测在"字符层"做，但模型在"Token 层"理解。Token boundary 是两层之间的映射缝隙，攻击者可以在此隐藏恶意内容。Transformer 用注意力机制替代了 RNN 的串行处理，让模型能并行计算——但这也意味着"位置信息"需要额外编码。理解 Token 如何进入 Transformer，是理解后续 Prompt Injection 攻击的基础。
