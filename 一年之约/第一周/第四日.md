## 写了什么

文件位置：`projects/p1/llm_security_eval.py`

两个核心类：

### LLMTarget — 发消息的工具

负责连接 AI 模型、发送攻击语句、拿回回复。

```python
class LLMTarget:
    def __init__(self, base_url, model, system_prompt):
        self.base_url = base_url
        self.model = model
        self.system_prompt = system_prompt

    def query(self, user_msg):
        # 发消息给模型，返回回复
        ...
```

**大白话**：就像"替你给 AI 发消息的小弟"。

### SecurityFinding — 记录本

负责记录每次攻击测试的结果。

```python
@dataclass
class SecurityFinding:
    attack_id:          str    # 攻击编号
    technique:          str    # 攻击技术名称
    principle_violated: str    # 违反了什么原理
    payload:            str    # 攻击语句
    response:           str    # 模型回复
    verdict:            str    # INJECTED / PARTIAL / DEFENDED
    asr:                float  # 攻击成功率
    owasp_category:     str    # OWASP 分类
    mitre_atlas:        str    # MITRE ATLAS 编号
```

**大白话**：就像"成绩单"，记录每次攻击 AI 有没有中招。

## 踩的坑（重要！）

### 坑1：Ollama 接口选错了

| 接口 | 结果 |
|------|------|
| `/v1/chat/completions` | ❌ 超时 |
| `/api/chat` | ❌ 3B 模型输出乱码 |
| `/api/generate` | ✅ 正常 |

**结论**：用 Ollama 原生 `/api/generate` 接口。

### 坑2：模型选错了

| 模型 | 结果 |
|------|------|
| llama3.2:3b | ❌ CPU 上输出乱码 "incident incident..." |
| llama3.1:8b | ✅ 正常输出，但慢（约 36 秒/次） |

**结论**：用 llama3.1:8b。

### 原因

电脑显卡太老（NVIDIA GT 730，2GB），Ollama 不支持 GPU 加速，只能用 CPU 跑。3B 模型在 CPU 上推理质量差，8B 反而正常。

## 两个类怎么配合

```
你写攻击语句
    ↓
LLMTarget 发给 AI 模型
    ↓
AI 回复
    ↓
SecurityFinding 记录结果
```

**就像玩游戏打怪**：
- LLMTarget = 武器（负责打怪）
- SecurityFinding = 战绩本（记录输赢）
